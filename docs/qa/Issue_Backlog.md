# Issue Backlog - UX / A11y / Responsive

Ce backlog regroupe les améliorations identifiées lors de l'audit systématique de l'interface et de l'accessibilité.

## 🔴 Critiques (Bloque l'usage optimal ou viole une règle stricte)

| ID | Page(s) | Breakpoint | Type | Description | Résultat Attendu |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **UXA11Y-001 (CLOS 2026-07-22)** | `P-001`, `P-003`, `P-005` | Mobile (<768px) | Responsive | Les tableaux de données (Leaderboard, KvK, Bank) forcent le scroll horizontal. **Obsolète** : les vues en cartes ont été livrées entre-temps sur les quatre pages concernées (`md:hidden` cartes / `hidden md:block` table). Mesuré sur staging à 375px le 2026-07-22 : aucun `<table>` n'est rendu sur `/`, `/kvk`, `/bank`, `/trophies`, `/war-tracker`. | Rien à faire. Deux débordements résiduels d'une autre nature ont été trouvés pendant la vérification et corrigés — voir UXA11Y-009. |
| **UXA11Y-002 (CORRIGÉ 2026-07-22)** | Toutes | Tous | A11y | Le constat d'origine (« aucune indication ») était trop large : une règle `:focus-visible` globale existait et fonctionnait sur les boutons ordinaires. Deux défauts réels mesurés en prod : l'anneau Tailwind passe par `box-shadow`, donc **`.v2-nav-item.act`** (entrée de nav active, sidebar et bottom nav) l'écrasait avec sa propre lueur et n'affichait aucun focus ; et l'offset était figé sur `ring-offset-slate-900`, faux en thème clair. | **Fait** : passage à un `outline` (immunisé contre les ombres de composants), couleur par token `--focus-ring` déclinée par thème. Vérifié au clavier sur staging, thèmes sombre et clair. |
