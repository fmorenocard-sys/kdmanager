# Kingdom Manager (KD 2997)

Community management dashboard for a Rise of Kingdoms kingdom: war attendance (War Tracker), KvK performance tracking, resource bank, deadweight monitoring, Discord SSO with role sync. Users: King (global config) > Officers (data updates, attendance) > Warriors (declare availability, view stats) > Guest.

## Stack

- **Frontend**: React 19 + Vite, Tailwind CSS (glassmorphism dark theme, Slate-900), react-router-dom (HashRouter), i18next (9 languages incl. RTL Arabic), Recharts, framer-motion, lucide-react
- **Backend**: Firebase — Firestore (two databases: `(default)` and `kdmanagerdb`), Cloud Functions (`functions/`, Node ESM), Firebase Auth + Discord OAuth2 (custom token minting in `functions/discordAuth.js`)
- **Data ingestion**: XLSX exports parsed client-side via SheetJS + `src/config/data-mapping.js`; static snapshots in `public/data/` regenerated by `scripts/digest-data.js` (runs as part of `npm run build`)
- **RBAC**: role docs in Firestore `roles` collection; enforced in `firestore.rules` (helpers `isOwner`/`isKingOrOfficer`) and in UI guards

## Commands

- `npm run dev` — dev server
- `npm run build` — digest data + vite build
- `npm run lint` — eslint
- `npm run test:e2e` — Playwright suite inside `firebase emulators:exec` (specs in `tests/`)
- `npm run test:serve` — dev server pointed at the Firebase emulator (port 5174)
- `node scripts/deploy-rules.js` — deploy `firestore.rules` to BOTH Firestore databases (plain `firebase deploy` only covers what `firebase.json` lists)

## UI conventions (from `.agent/rules/responsiveness.md`)

- Breakpoints: 360px mobile / 768px tablet / 1024px+ desktop. **No horizontal page scroll, ever** — wide tables get mobile card views or their own scroll container.
- Use logical properties (`start`/`end`, `ms-`/`me-`) not `left`/`right` — Arabic RTL is supported.
- Tap targets ≥ 44×44px; visible `:focus` rings; WCAG AA contrast; `<button>` for actions, `<a>` for navigation.
- Every user-facing string goes through i18next — add the key to **all** locale files in `src/locales/*/translation.json` AND `public/locales/*/translation.json` (fr, en, es, de, ar, pl, tr, uk, vi).

## Security — hard rules (incident 2026-07-11)

Secrets were once committed because a corrupted `.gitignore` entry silently failed; history had to be scrubbed. Do not repeat it:

- **Never track**: `.env*` (except `.env.example`), `functions/serviceAccountKey.json`, `functions/service-account.json`, any service-account key, `users.json` / `functions/all_users*.json` (real user PII), captured logs (`logs*`, `*_logs.txt`) which may contain OAuth codes.
- **Never hardcode secrets in source** — Cloud Functions use `defineSecret()` (see `functions/discordAuth.js`); scripts read local ADC credentials.
- Before any commit that adds new files, scan them for embedded credentials.
- Write `.gitignore` and other dot-files as UTF-8 (PowerShell `Out-File` defaults to UTF-16 and corrupts them — use `-Encoding utf8` or the Write tool).
- One-off scratch scripts/dumps go in `scratch/` or `tmp/` (gitignored), not the repo root.

## Design system (Claude Design)

The UI kit is mirrored to the "KD Manager Design System" project on claude.ai/design (projectId `d5cd9187-40ba-45fc-a2f4-b6ac5f946c04`). Source of truth stays in code: `design-system/generate.mjs` builds self-contained HTML cards into `design-system/cards/` from the real tokens (`src/index.css` @theme, `src/components/ui/*`). After changing UI tokens or components, re-run the generator and re-sync the affected cards via DesignSync (incremental, never wholesale). Note: primary buttons are **amber-500**, not indigo — the indigo family is for active tabs/current badges.

## Known technical debt (see `docs/project_context.md` §5)

- XLSX ingestion is tightly coupled to the exact export format; header/tab changes require `src/config/data-mapping.js` + `functions/data-mapping.js` maintenance (keep both in sync).
- Leaderboard loads the full `player_data` set — watch Firestore read costs; prefer aggregations.
- Discord OAuth token parsing works around HashRouter's `#` handling — fragile, touch with care.
